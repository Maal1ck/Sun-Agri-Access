Workflow Technique

Découvrez l'architecture des données et les algorithmes utilisés pour calculer la rentabilité de l'irrigation solaire au Sénégal.

PHASE 01

Acquisition des Données

Collecte des données brutes multisources pour alimenter le modèle. Nous croisons des données hydrogéologiques locales avec des données satellitaires mondiales.

> INPUT: Forages DGPRE (CSV)
> INPUT: Global Solar Atlas (GeoTIFF)
> INPUT: ESA WorldCover (Sentinel-2)
PHASE 02

Interpolation Spatiale (IDW)

Les données de forages sont ponctuelles. Pour obtenir une carte continue de la profondeur de la nappe sur tout le territoire, nous utilisons l'algorithme IDW (Inverse Distance Weighting) dans QGIS.

Processing: QGIS Raster Calc
Output: Raster Continu (Resolution 100m)
PHASE 03

Modélisation Économique

C'est le cœur du système. Pour chaque pixel, nous calculons l'énergie requise pour pomper 50m³/jour, puis nous dimensionnons le système solaire nécessaire et son coût sur 20 ans.

FORMULE LCOW :
(Puissance_Crete * Prix_Wc) / (Volume_Total)
PHASE 04

Déploiement WebGIS

Les résultats sont vectorisés (Hexbins) pour être légers et interactifs. L'interface finale est propulsée par Kepler.gl et hébergée sur GitHub Pages pour un accès universel.

Stack: HTML5 / Kepler.gl / Mapbox
Hosting: GitHub Pages
